#### 행렬(Matrix)

행렬에서 가로줄은 행(row), 세로줄은 열(column)이다.



행렬은 좌표를 어떻게 변환할지를 정의한 변환 규칙을 수식으로 표현한 구조이다.

점 하나 (x, y, z)가 있을 때,

이 점을 어디로 이동시키고, 얼마나 회전시키고, 얼마나 확대 축소 할지에 대한

규칙을 숫자로 표현한 것이 행렬이다.



즉,행렬은 좌표 변환에 대한 수학적 표현식이다.



#### 역행렬(Inverse Matrix)

역행렬은 어떤 변환 행렬을 적용하기 이전 상태로 되돌리는 변환 행렬이다.

즉, 어떤 행렬 A가 있을 때,

A \* A⁻¹ = I

가 되며,

이는 변환을 적용하기 전 좌표 상태로 되돌린다는 의미이다.

개념적으로는 되돌리기 버튼 컨트롤 Z과 같은 역할을 한다.



#### 이동(Translation)

이동은 좌표의 기준 위치를 바꾸는 변환이다.



어떤 점이 (x, y, z)에 있을 때,

이동 변환은 이 점을 같은 방향과 형태를 유지한 채 다른 위치로 옮긴다.

이동 변환은 이 점을 같은 방향과 형태를 유지한 채 다른 위치로 옮긴다.

* 점의 모양이나 방향은 변하지 않는다.
* 모든 정점이 같은 거리만큼 동일하게 이동한다.
* 객체 전체가 공간안에서 통째로 밀리는 느낌이다.



이동은 "이 객체가 세상에서 어디에 존재하는가"를 결정하는 변환이다.

#### 크기

크기 변환은 좌표를 기준점으로부터 얼마나 멀리 떨어뜨릴지를 조절하는 변환이다.

* 중심점을 기준으로
* 각 축 방향으로 거리를 늘리거나 줄인다
* 값이 커질수록 정점은 기준점에서 더 멀어지고
* 값이 작아질수록 정점은 기준점에 가까워진다



크기 변환은 "객체가 기준점에 대해 얼마나 커지거나 작아지는가"를 정의한다.

중요한 점은

* 위치 자체를 옮기는 것이 아니라
* 기준점과의 거리 비율을 바꾸는 것이라는 점이다.



#### 회전

회전은 기준점을 중심으로 좌표의 방향을 바꾸는 변환이다.

* 좌표는 같은 거리만큼 떨어진 상태를 유지하고
* 방향만 바뀐다.
* 즉,크기는 변하지 않는다

회전은 항상 어떤 축과 기준점을 기준으로 이루어진다.



회전변환은 "객첵ㅏ 어느 방향을 바라보고 있는가"를 결정한다.



#### 왜 이 세가지가 항상 함께 쓰이는가?

이동, 크기, 회전은 각각 다른 역할을 가진다.

* 크기 : 객체의 형태 비율
* 회전 : 객체의 방향
* 이동 : 객체의 위치

이 세가지를 조합해야만 하나의 객체를 공간안에 정확하게 배치할 수 있다.



그래서 그래픽스에서는 이 세 변환을

하나의 변환 행렬로 결합하여 사용한다.



#### 변환 행렬 곱셈 순서

객체 기준(Local Space) 변환 순서는 다음과 같다.

Scale -> Rotation -> Translation

크기 -> 자전 -> 이동 -> 공전 ->  부모

* 크기 조절
* 회전
* 이동

순서로 변환이 적용된다.



그래픽스 파이프라인 전체 좌표계 흐름은 다음과 같다.

Local -> World -> View -> Projection



#### 변환 순서가 중요한 이유

변환은 적용 순서에 따라 결과가 달라진다.

객체 기준(Local Space)에서는 일반적으로 다음 순서를 따른다.

1. ###### 크기
2. ###### 회전
3. ###### 이동

이 순서는 다음 의미를 가진다.

* 먼저 객체 자체의 크기를 정하고
* 그 상태에서 방향을 회전시키고
* 마지막으로 공간상의 위치로 이동시킨다

이 순서를 바꾸면 회전하면서 이동하는 객체 처럼 전혀 다른 결과가 나온다.



#### 공전과 부모 변환의 의미

객체가 다른 객체의 자식일 경우

* 자식의 Local 변환 결과는
* 부모의 변환을 기준으로 다시 한 번 변환된다

이때 자식이 부모를 중심으로 움직이는 것처럼 보이는데,

이를 공전이라고 이해할 수 있다.



실제로는 공전이라는 변환이 따로 있는 것이 아니라

부모 변환이 먼저 적용되기 때문에 그렇게 보이는 것이다.
