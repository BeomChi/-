### 데크(Dequeue)

데크(Deque)는 양쪽 끝(front,back)에서 삽입과 삭제가

가능한 자료구조이다.

즉,큐(Queue)와 스택(Stack)의 특징을 모두 가진다.



앞에서도 삽입/삭제 가능

뒤에서도 삽입/삭제 가능



### 데크 특징

양쪽 끝에서 삽입/삭제 가능

중간 접근은 비효울적

FIFO와 LIFO 모두 구현가능

스택과 큐를 일반화한 구조



### 기본연산

| 연산         | 설명       |

| ---------- | -------- |

| push\_front | 앞에 삽입    |

| push\_back  | 뒤에 삽입    |

| pop\_front  | 앞에서 삭제   |

| pop\_back   | 뒤에서 삭제   |

| front      | 앞 데이터 확인 |

| back       | 뒤 데이터 확인 |

| empty      | 비어있는지    |

| size       | 크기       |



##### C++ STL
```
deque<int> dq;



dq.push\_front(10);

dq.push\_back(20);



dq.pop\_front();

dq.pop\_back();



dq.front();

dq.back();
```


### 시간 복잡도

| 연산         | 시간   |

| ---------- | ---- |

| push\_front | O(1) |

| push\_back  | O(1) |

| pop\_front  | O(1) |

| pop\_back   | O(1) |

| 접근 (index) | O(1) |



### 내부 구현 방식

##### 원형 배열(Circular Array)

queue처럼 head/tail 사용

양쪽에서 움직임



##### 블록 배열 구조(STL deque)

\[블록]\[블록]\[블록]

각 블록은 연속된 배열

블록끼리는 포인터로 연결



### 특징

* vector처럼 한 번에 크게 복사 안함
* 앞뒤 확장 빠름
* 메모리 단편화 적음



### Vector vs Deque

| 구분    | Vector | Deque |

| ----- | ------ | ----- |

| 메모리   | 연속     | 블록 단위 |

| 앞 삽입  | O(n)   | O(1)  |

| 뒤 삽입  | O(1)   | O(1)  |

| 중간 삽입 | O(n)   | O(n)  |

| 캐시 효율 | 좋음     | 보통    |



앞쪽에도 삽입이 필요하다 그럼 deque

뒤에만 삽입하는걸로 충분하다 그럼 vector



### 언제 사용하는가

##### 양쪽에서 삽입/삭제가 필요한 경우

슬라이딩 윈도우

양방향 탐색



##### 우선 순위 구조

큐 + 우선순위 일부 -> deque

최근 데이터 우선 처리(LIFO)

오래된 데이터도 유지(FIFO)



##### BFS 변형 (0-1 BFS)

가중치가 0 또는 1일 때

0 -> push\_front

1 -> push\_back

이게 다익스트라보다 빠르다고 한다.

##### 작업 스케줄링

최근 작업 우선

오래된 작업도 유지



### 스택 큐 데크 관계 정리

Stack -> 한쪽만 사용

Queue -> 한쪽 넣고, 반대쪽 빼기

Deque -> 양쪽 다 사용



### 정리

Deque는 양쪽 끝에서 삽입과 삭제가 가능한 자료구조로,

스택과 큐의 기능을 모두 포함한다.

슬라이딩 윈도우, 0-1 BFS, 양방향 처리 문제에서 주로 사용된다.
