### AVL 트리

스스로 균형을 잡는 이진 탐색 트리이다.

트리의 높이가 h일 때 이진 탐색 트리의 시간 복잡도는 O(h)이다.

한쪽으로 치우친 편향 이진트리가 되면 트리의 높이가 높아지 떄문에

이를 방지하고자 높이 균형을 유지하는 AVL 트리를 사용하게 된다.



### 특징

AVL 트리는 이진 탐색 트리의 속성을 가진다.

왼쪽, 오른쪽 서브 트리의 높이 차이가 최대 1이다.

어떤 시점에서 높이 차이가 1보다 커지면 회전(rotation)을 통해 균형을 잡아 높이 차이를 줄인다.

AVL 트리는 높이를 LogN으로 유지하기 때문에 삽입, 검색 ,삭제의 시간 복잡도는 O(log N)이다.



### 시간 복잡도

|연산|시간 복잡도|
|-|-|
|검색|O(log N)|
|삽입|O(log N)|
|삭제|O(log N)|



트리 높이를 항상 log N으로 유지하기 떄문이다.



### Balance Factor

Balance Factor는 왼쪽 서브트리의 높이에서 오른쪽 서브트리의 높이를 뺀 값이다.



```

Balance Factor(k) = hegiht(left(k)) - height(right(k))

```

### BF 값 의미

BF가 1이면 왼쪽 서브트리가 오른쪽 서브트리보다 높이가 한단계 높다는 것을 의미한다.

BF가 0이면 왼쪽 서브트리가 오른쪽 서브트리의 높이가 같다는 것을 의미한다.

BF가 -1이면 왼쪽 서브트리가 오른쪽 서브트리보다 높이가 한단계 낮다는 것을 의미한다.



정상 범위

```

\-1 <= BF <= 1

```

이 범위를 벗어나면

```

불균형 발생 -> 회전 필요

```





### 회전

삽입 또는 삭제 후 균형이 깨지면

트리 구조를 바꿔서 균형을 맞춘다.



AVL 트리는 이진 탐색 트리이기 때문에 모든 작업은 이진 탐색 트리에서 사용하는 방식으로 수행한다.

검색 및 순회 연산은 BF를 변경하지 않지만 삽입 및 삭제에서는 BF가 변경될 수가 있다.

삽입 삭제 시 불균형 상태(BF가 -1,0,1이 아닌 경우)가 되면 AVL 트리는 불균형 노드를

기준으로 서브트리의 위치를 변경하는 회전 작업을 수행하여 균형을 맞춘다.



삽입 삭제시 노드들의 배열에 따라 4가지(LL,RR,LR,RL) 불균형이 발생할 수 있으며 각 상황마다

회전에 방향을 달리하여 트리의 균형을 맞춘다.



#### 회전 종류

##### LL

왼쪽의 왼쪽에 삽입됨

해결 : 오른쪽 회전

```

&#x20;   A

&#x20;  /

&#x20; B

&#x20;/

C

```



##### RR

오른쪽의 오른쪽에 삽입됨

해결 : 왼쪽 회전

```

A

&#x20;\\

&#x20; B

&#x20;  \\

&#x20;   C

```

##### LR

왼쪽의 오른쪽에 삽입됨

해결

```

1\. Left Rotation(B 기준)

2\. Right Rotation(A 기준)

```

```

&#x20;   A

&#x20;  /

&#x20; B

&#x20;  \\

&#x20;   C

```

##### RL

오른쪽의 왼쪽에 삽입됨

해결

```

1\. Right Rotation(B 기준)

2\. Left Rotation(A 기준)

```



```

A

&#x20;\\

&#x20; B

&#x20;/

C

```



##### 흐름

BST 처럼 삽입 / 삭제 수행

BF 계산

|BF| > 1 -> 불균형

LL / RR / LR / RL 판단

회전 수행



### BST VS AVL

|구분|BST|AVL|
|-|-|-|
|균형|없음|유지|
|최악 시간|O(N)|O(log N)|
|사용 목적|단순 구조|성능 보장|



### 정리

AVL 트리는 균형을 유지하는 이진 탐색 트리로,

모드 노등ㅔ서 높이 차이를 1이하로 유지한다.



균형이 깨지면 회전을 통해 복구하며,

항상 O(log N)의 성능을 보장한다.
