# 그래프(Graph)



### 그래프 개념

그래프(Graph)는 여러 개의 정점(Vertex)과 간선(Edge)으로 이루어진 자료구조로,

정점들 사이의 관계를 표현하는 데 사용된다.



### 트리와 그래프의 차이점

* 트리

  * 루트 노드가 존재
  * 사이클이 없음
  * 부모 - 자식 관계 
  * 계층구조
* 그래프

  * 루트 노드 개념 없음
  * 사이클 존재 가능
  * 부모 - 자식 관계가 아닌 정점 간의 관계 표현
  * 일반적인 연결 구조


즉, 그래프는 계층 구조가 아니라 관계 구조를 표현한다.

예
* 지도 길 찾기
* SNS 친구 관계
* 네트워크 연결
* 게임 맵 이동
* 컴파일러 의존성 그래프



### 그래프 용어



##### 정점(Vertex)

그래프에서 하나의 점(노드)을 의미한다.

데이터 하나를 저장하는 단위이다.

예

* 사람
* 도시
* 컴퓨터
* 게임에서 위치 노드

Vertex = Node 같은 의미로 사용됨.



##### 간선(Edge)

정점과 정점을 연결하는 선

즉, 정점 간의 관계를 나타낸다.

예

* 사람 <-> 친구 관계
* 도시 <-> 도로
* 컴퓨터 <-> 네트워크 연결



##### 인접 정점(Adjacent vertex)

하나의 정점에서 간선에 의해 직접적으로 연결되어 있는 정점

예

A -> B 연결되어 있으면

B는 A의 인접 정점



##### 가중치 그래프

간선에 비용,거리,시간 같은 값을 부여한 그래프

예

* 길 찾기 -> 거리
* 네트워크 -> 비용
* 게임 -> 이동 비용

사용 알고리즘
* 다익스트
* 최소 스패닝 트리
* 플로이드 워셜

다익스트라 알고리즘,최소 스패닝 트리에서 사용됨.



##### 차수

###### 무방향 그래프

정점에 인접한 정점의 수



###### 방향 그래프

* 외부에서 들어오는 간선의 수를 뜻하는 진입차수
* 외부로 향하는 간선의 수를 뜻하는 진출 차수



##### 길이

정점과 다른 정점의 경로를 구성하는 데 사용된 간선의 수



##### 단순 경로

경로 중에서 정점을 한 번씩만 방문하는 경로

즉, 같은 정점을 다시 방문하지 않음



##### 사이클

시작 정점과 끝 정점이 같은 경로

예

A -> B -> C -> A

그래프에서 매우 중요한 개념

(유니온 파인드, 위상 정렬, MST 등에서 사용)



### 그래프 종류



##### 무방향 그래프

간선에 방향이 없음

양쪽 이동 가능

A - B

A < - > B

예

* 일반통행 도로
* 작업 순서
* 의존성 그래프



### 그래프의 특징



##### 그래프 특징

1. 루트 노드가 없다
2. 부모 - 자식 관계가 없다.
3. 사이클이 존재할 수 있다.
4. 정점 수 만큼 간선이 존재하지 않아도 된다.
5. 하나의 정점에서 여러 간선이 연결될 수 있다.
6. 순회를 위해 DFS,BFS 사용
7. 트리도 그래프의 일종이다.( 사이클 없는 그래프)

### 그래프 표현 방식



##### 인접 리스트 (Adjacency List)

연결된 정점만 저장하는 방식



###### 특징

* 메모리 적게 사용
* 대부분 알고리즘 문제에서 사용
* DFS / BFS에 매우 적합
* 간선이 적은 그래프에 유리



###### 시간 복잡도

* 특정 정점에서 연결된 정점 탐색 : O(연결된 간선 수)
* 두 정점 연결확인 : O(연결된 간선 수)

##### 인접행렬(Adjacency Matrix)

정점 연결 여부를 2차원 배열로 표현



###### 특징

* 메모리 많이 사용 O(V^2)
* 두 정점 연결 확인 매우 빠름 O(1)
* 간선이 많은 그래프에 적합



###### 시간 복잡도

* 연결 확인 O(1)
* 전체 탐색 O(V^2)

### 인접 리스트 VS 인접 행렬

| 구분     	 | 인접 리스트 | 인접 행렬 |

| ------------ | ------------- | ----------- |

| 메모리    	  | 적음	    | 많음   	    |

| 연결 확인   | 느림  	    | 빠름    	    |

| 전체 탐색   | 빠름     	    | 느림    	    |

| 간선 적음   | 좋음          | 비효율      |

| 간선 많음   | 보통     	    | 좋음    	    |

| DFS/BFS 	  | 매우 좋음   | 보통         |



##### 정리

간선 적다 -> 인접 리스트

간선 많다 -> 인접 행렬

연결 확인 많이 한다 -> 인접 행렬

그래프 탐색 -> 인접 리스트



### 그래프에서 사용하는 알고리즘

1. DFS (깊이 우선 탐색)				그래프 탐색
2. BFS (너비 우선 탐색)				최단 거리(가중치 없음)
3. 다익스트라					최단 거리(가중치 있음)
4. 최소 스패닝 트리 				(Kruskal, Prim)		크루스칼, 프림
5. 위상 정렬					작업 순서
6. 유니온 파인드					사이클 판별
7. 플로이드 워셜					모든 정점 최단거리



그래프 = DFS / BFS



### 한 줄 정리

그래프 = 정점 ( Vertex) + 간선 (Edge)



그래프 표현 방법

1. 인접 리스트 -> 메모리 적게, DFS/ BFS
2. 인접 행렬 -> 연결 확인 빠름

그래프 종류
- 방향 그래프
- 무방향 그래프
- 가중치 그래프

그래프 표현
- 인접 리스트
- 인접 행렬

- DFS
