#### 프로세스(Process)

프로세스는 실행중인 프로그램의 단위이다.



프로세스가 독립된 메모리를 갖는 이유는 안정성 때문이다. 한 프로세스가

오류로 인해 비정상 종료되더라도, 독립된 공간을 쓰기 때문에 다른 프로세스에

직접적인 영향을 주지 않는다.



* 운영체제로부터 독립된 메모리 공간을 할당받음
* 코드, 데이터, 힙, 스택을 포함
* 다른 프로세스와는 기본적으로 메모리를 공유하지 않음



프로그램이 실행되면 하나의 프로세스가 된다.



#### 스레드(Thread)

스레드는 하나의 프로세스 내부에서 실행되는 작업 단위이다.

* 같은 프로세스 안의 스레드들은

 	코드

 	데이터

 	힙

 	을 공유한다

각 스레드는 자기만의 스택과 레지스터를 가진다



공유하는 이유 :  같은 작업을 하는 일꾼들이 굳이 자료를 복사해서 따로 가질 필요 없이,

공동 작업대(Heap, Data)를 쓰는 것이 효율적이기 때문이다.



스택을 따로 갖는 이유 : 스레드는 독립적인 함수 호출(실행 흐름)을 가져야 한다.

어디까지 실행했는지, 함수 안의 지역 변수는 무엇인지 는 일꾼마다 달라야 하므로

독립된 Stack이 필수이다.



한 프로그램안에서 실제로 일을 수행하는 실행 흐름



프로세스 간 전환보다 스레드 간 전환이 훨씬 바르다.

공유하는 자원이 많아 바꿀 정보가 적기 때문이다.



결국 프로세스는 회사나 조직이며 스레드는 회사나 조직에서 일하는 일꾼이다.
