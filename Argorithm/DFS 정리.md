### DFS 정리



#### DFS란 무엇인가

DFS는 그래프 또는 트리에서 사용하는 탐색 방법으로,

한 방향으로 갈 수 있을 때까지 계속 깊게 파고 들어가고

더 이상 갈 수 없으면 이전 지점으로 되돌아와 다른 길을 탐색하는 방식



한 문장으로 정리하면 DFS는 한 길을 끝까지 파고 들어가는 탐색 방법이다.





#### DFS 탐색 과정

A

├─ B

│  ├─ D

│  └─ E

└─ C

&#x20;  └─ F

예를 들어 이런 그래프가 있다고 가정을 해보자



그럼 DFS 탐색 순서는

A -> B -> D -> (끝) -> B -> E -> (끝) -> A -> C-> F

로 탐색을 하게 된다

끝까지 갔다가 -> 되돌아 온다 -> 다른 길을 간다

이 되돌아오는 과정이 DFS의 핵심이다.

이걸 Backtrackin 이라고 한다.



#### 왜 DFS는 스택 또는 재귀를 사용할까

DFS의 동작을 보면 

A → B → D

&#x20;       ↑

&#x20;  여기서 막힘

&#x20;  → B로 돌아감

즉 우리는 항상 이전 있었던 위치로 돌아가야 한다는 것이다.

이 동작 구조가 자료구조로 보면 마지막에 들어간 위치부터 꺼내는 LIFO(Last In First Out)

구조인 Stack 자료구조가 있다



그래서 정리하면 

DFS는 되돌아가기 위해 이전 경로를 저장해야 하고

이 구조가 Stack과 동일하기 때문에 Stack 또는 재귀함수로 구현한다.



#### 재귀로 DFS가 되는 이유

재귀 함수 내부적으로는 이렇게 동작한다

dfs(A)

&#x20;└ dfs(B)

&#x20;    └ dfs(D)

함수가 호출될 때마다 이전 함수 위치가 스택에 저장된다.

그래서 D 끝 -> 함수 종료 -> 자동으로 B로 돌아감

즉

재귀함수 = 내부적으로 Stack 사용

그래서

DFS

&#x20;= Stack 구현

&#x20;= 재귀함수 구현



#### 재귀로 구현한 DFS 예제코드

void dfs(int cur)

{

&#x20;   visited\[cur] = true;



&#x20;   for(int next : graph\[cur])

&#x20;   {

&#x20;       if(!visited\[next])

&#x20;           dfs(next);

&#x20;   }

}



for(int i = 1; i <= N; i++)

{

&#x20;   if(!visited\[i])

&#x20;       dfs(i);

}

#### DFS 시간 복잡도

DFS 시간 복잡도: O(V + E)



V = 정점 개수

E = 간선 개수



모든 노드 한 번 방문

모든 간선 한 번 확인



그래서 O(V + E)가 된다.



#### DFS 및 BFS 차이

| 구분  	| DFS         		| BFS     		 |

| -----------	| ------------------------	| ------------------	 |

| 탐색 방식 | 깊이 우선       		| 가까운 곳 우선	 |

| 자료구조  | Stack       			| Queue    		 |

| 경로    	| 한 방향 끝까지 		| 레벨 순서    	 |

| 최단거리  | 보장 X        		| 보장 O     	 |

| 구현    	| 재귀         		| 큐        		 |

| 사용   	| 경로 존재, 백트래킹 	| 최단거리   	 |



| 문제 유형		| DFS | BFS |

| -----------------	| --- 	| --- |

| 모든 경우의 수  | O   	| X   |

| 최단 거리  	| X   	| O  |

| 사이클 탐지   	| O  	| O  |

| 연결 요소   	| O  	| O  |

| 백트래킹     	| O  	| X   |

| 레벨 탐색    	| X   	| O  |



| 알고리즘    | visited 처리 시점  		|

| -----------	  | ----------------------------	|

| DFS   	  | 함수 들어가자마자   		|

| BFS   	  | queue에 넣을 때    		|

| 다익스트라 | 더 짧은 거리 갱신될 때   	|

| A\*    	  | open list 들어갈 때 		|



DFS는 깊이를 우선으로 탐색하는 알고리즘이고

BFS는 거리를 기준으로 탐색하는 알고리즘이다.

깊이 / 경우의 수 → DFS

최단거리 / 최소 이동 → BFS



#### DFS는 언제 사용하는가

DFS가 자주 쓰이는 경우

1\. 모든 경우의 수 탐색 (완전탐색)

2\. 경로 존재 여부 확인

3\. 사이클 탐지

4\. 백트래킹 문제

5\. 트리 탐색

6\. 연결 요소 찾기



대표 문제 유형

* 미로 탐색
* 조합
* 순열
* N-Queen
* 부분집합
* 그래프 연결 요소



#### 결론 총 정리

1\. 모든 경우의 수를 탐색해야 한다 → DFS

2\. 최단 거리를 구해야 한다 → BFS

3\. 가중치가 있는 최단 거리 → 다익스트라

4\. 더 빠른 경로 탐색 (휴리스틱) → A\*

5\. 연결된 그룹 개수 → DFS 또는 BFS

6\. 사이클 탐지 → DFS 또는 BFS



##### 문장 버전

DFS(Depth First Search)는 그래프 탐색 방법 중 하나로,

한 방향으로 갈 수 있을 때까지 계속 깊게 탐색하고,

더 이상 갈 수 없으면 이전 노드로 되돌아와 다른 경로를 탐색하는 방식이다.

이 되돌아가는 과정 때문에 DFS는 스택 구조를 사용하며,

재귀 함수는 내부적으로 스택을 사용하기 때문에 DFS를 재귀로 구현할 수 있다.

DFS에서는 무한 루프를 방지하기 위해 노드에 들어가는 순간 visited 처리를 해야 하며,

인접 노드를 탐색하면서 방문하지 않은 노드만 재귀적으로 탐색한다.

DFS는 모든 경우의 수 탐색, 백트래킹, 사이클 탐지, 연결 요소 탐색 등에 자주 사용된다.
