DFS 정리

DFS란 무엇인가
DFS는 그래프 또는 트리에서 사용하는 탐색 방법으로,
한 방향으로 갈 수 있을 때까지 계속 깊게 파고 들어가고
더 이상 갈 수 없으면 이전 지점으로 되돌아와 다른 길을 탐색하는 방식

한 문장으로 정리하면 DFS는 한 길을 끝까지 파고 들어가는 탐색 방법이다.


DFS 탐색 과정
A
├─ B
│  ├─ D
│  └─ E
└─ C
   └─ F
예를 들어 이런 그래프가 있다고 가정을 해보자

그럼 DFS 탐색 순서는
A -> B -> D -> (끝) -> B -> E -> (끝) -> A -> C-> F
로 탐색을 하게 된다
끝까지 갔다가 -> 되돌아 온다 -> 다른 길을 간다
이 되돌아오는 과정이 DFS의 핵심이다.
이걸 Backtrackin 이라고 한다.

왜 DFS는 스택 또는 재귀를 사용할까
DFS의 동작을 보면 
A → B → D
        ↑
   여기서 막힘
   → B로 돌아감
즉 우리는 항상 이전 있었던 위치로 돌아가야 한다는 것이다.
이 동작 구조가 자료구조로 보면 마지막에 들어간 위치부터 꺼내는 LIFO(Last In First Out)
구조인 Stack 자료구조가 있다

그래서 정리하면 
DFS는 되돌아가기 위해 이전 경로를 저장해야 하고
이 구조가 Stack과 동일하기 때문에 Stack 또는 재귀함수로 구현한다.

재귀로 DFS가 되는 이유
재귀 함수 내부적으로는 이렇게 동작한다
dfs(A)
 └ dfs(B)
     └ dfs(D)
함수가 호출될 때마다 이전 함수 위치가 스택에 저장된다.
그래서 D 끝 -> 함수 종료 -> 자동으로 B로 돌아감
즉
재귀함수 = 내부적으로 Stack 사용
그래서
DFS
 = Stack 구현
 = 재귀함수 구현

DFS 및 BFS 차이
| 구분  	| DFS         		| BFS     		 |
| -----------	| ------------------------	| ------------------	 |
| 탐색 방식 | 깊이 우선       		| 가까운 곳 우선	 |
| 자료구조  | Stack       			| Queue    		 |
| 경로    	| 한 방향 끝까지 		| 레벨 순서    	 |
| 최단거리  | 보장 X        		| 보장 O     	 |
| 구현    	| 재귀         		| 큐        		 |
| 사용   	| 경로 존재, 백트래킹 	| 최단거리   	 |


DFS는 언제 사용하는가
DFS가 자주 쓰이는 경우
1. 모든 경우의 수 탐색 (완전탐색)
2. 경로 존재 여부 확인
3. 사이클 탐지
4. 백트래킹 문제
5. 트리 탐색
6. 연결 요소 찾기

대표 문제 유형
미로 탐색
조합
순열
N-Queen
부분집합
그래프 연결 요소

결론 총 정리
1. 한 방향으로 끝까지 탐색한다
2. 막히면 이전 지점으로 되돌아간다
3. 되돌아가기 위해 Stack 또는 재귀를 사용한다
4. 모든 경우의 수 탐색에 많이 사용된다
