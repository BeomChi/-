#### 함수 포인터(Function Pointer)

함수 포인터는 함수의 주소를 저장하는 포인터이다.

이를 통해 런타임에 호출할 함수를 유연하게 변경할 수 있으며,

주로 콜백 함수나 C 스타일 인터페이스에서 활용된다.

* 함수 시그니처(반환형, 매개변수)가 정확히 일치해야한다
* 함수 자체를 가리키므로 상태를 가질 수 없다.
* 비교적 가볍지만, 표현력이 제한적이다.



#### 함수객체(Function Object / Functor)

함수 객체는 operator()를 오버로딩한 클래스 또는 구조체의 객체로,

함수처럼 호출할 수 있으면서 내부 상태를 유지할 수 있다.

* 멤버 변수를 통해 상태(state)를 저장 가능
* 함수 포인터보다 유연하고 확장성이 높음
* STL 알고리즘(std::sort, std::find\_if 등)에서 자주 사용됨



#### 함수 포인터와 함수 객체의 차이점

함수 객체는 operator()를 오버로딩한 객체로,

함수 포인터와 달리 내부 상태를 가질 수 있으며

더 높은 표현력과 확장성을 제공한다



C++ 환경에서 많은 경우 함수 포인터를 함수 객체로 대체할 수 있지만,

C API 연동이나 함수 주소가 직접 필요한 상황에서는

함수 포인터가 여전히 필요하다.



#### std::function 의 필요성

std::function은 특정 함수 시그니처를 만족하는 모든 호출 가능한 객체(callable)를

하나의 타입으로 저장할 수 있는 C++ 표준 라이브러리 클래스이다.

* 함수 포인터
* 함수 객체(Functor)
* 람다 표현식
* std::bind로 생성된 객체

등을 모두 저장할 수 있다.

이를 통해 콜배 인터페이스를 통일할 수 있으며,

내부적으로는 타입 소거 기법을 사용해

서로 다른 타입의 호출 객체를 동일한 방식으로 관리한다.



* std::funtion은 유연한 대신 함수 포인터보다 느릴 수 있고

&nbsp;	동적할당이 발생할 수 있다

* 성능이 중요한 구간(게임 루프, 렌더 루프)에서는 직접 함수

&nbsp;	객체나 템플릿 전달이 더 적합할 수 있다.
