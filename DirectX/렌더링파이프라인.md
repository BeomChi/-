#### IA(Input Assembler)

정점 데이터를 GPU 파이프라인으로 조립해서 넘기는 단계

* 버텍스 버퍼, 인덱스 버퍼를 읽음
* 입력 레이아웃을 기준으로

 	POSITION

 	NORMAL

 	TEXCOORD

 	등을 구조화

* 프리미티브 타입 결정

 	삼각형

 	라인

 	포인터



버텍스를 해석해서 넘기는 단계



#### VS(Vertex Shader)

각 정점을 개별적으로 처리하는 단계

* 좌표 변환

 	Local -> World -> View -> Projection

* 정점 단위 연산
* 조명 계산의 일부 가능
* 출력

 	클립 공간 좌표

 	필셀 쉐이더로 전달된 보간 데이터



3D  -> 2D로 가기위한 변환의 핵심 단계





#### RS(Rasterizer)

정점으로 이루어진 도형을 픽셀 단위 데이터로 변환하는 단계

* 클리핑(화면 밖 제거)
* 뷰포트 변환
* 삼각형 -> 픽셀 후보(fragment) 생성
* 컬링(앞면 / 뒷면 제거)
* 깊이 값 생성



단순히 픽셀 후보 생성뿐만 아니라,보간이라는 중요한 작업도 같이 수행한다.

정점 셰이더에서 나온 정점 데이터(uv, 색상 등)을 픽셀들 사이의 거리만큼 수학적으로 계산해서

채워주는 과정이다.

기하 -> 픽셀의 경계 단계





#### PS(Pixel Shader)

각 픽셀의 최종 색상을 계산하는 단계

* 텍스처 샘플링
* 조명 계산
* 알파 처리
* 픽셀 단위 연산



이 픽셀이 무슨 색을 뛸지에 대해 결정





#### OM(Output Merger)

픽셀 쉐이더 결과를 기준으로 기존 렌더 타겟과 병합하는 단계

* 깊이 테스트
* 스텐실 테스트
* 블렌딩
* 렌더 타겟에 기록



깊이/ 스텐실 테스트를 통해 최종적으로 어떤 픽셀이 화면에 남을지 결정하는 '최종 판정돤' 역할을 한다

출력 직전 단계
